# LLMs.txt Design Guidelines

## Purpose

These files provide LLM-readable documentation indexes to help AI assistants quickly understand and reference this documentation site. They follow the [llms.txt specification](https://llmstxt.org/) and are optimized for Claude and other LLMs.

## Site Metadata

- **Title**: AI Tutorial.dev
- **URL**: https://aitutorial.dev
- **Source Code**: https://github.com/ai-tutorial/website
- **Examples Repo**: https://github.com/ai-tutorial/typescript-examples
- **Description**: An Interactive AI Tutorial with Concepts and Lessons Learned. Learn by experimenting with real code in an interactive sandbox. No setup, no dependencies—just open your browser and start building. Every concept includes runnable examples across multiple frameworks, so you can try things immediately and see what actually works.

## File Specifications

### llms.txt (Concise Version)

**Purpose**: Quick context loading — enough for an LLM to understand the site structure, find relevant sections, and answer questions without reading full content.

**Target size**: ~50KB

**Content structure**:
```markdown
# AI Tutorial.dev - LLM-Friendly Documentation Index
> [Site description]
> Last Updated: [git commit hash] [timestamp]
> Site: https://aitutorial.dev

## Table of Contents
[Hierarchical listing with absolute URLs]

## Module Name
Summary: [2-3 sentences — what it covers and why it matters]
Key Topics: [bullet points, 3-7 items]

### Section Name
[1-2 sentence description]
```

**Guidelines**:
- Keep module summaries to 2-3 sentences
- List only key topics (3-7 items per section)
- Focus on "what" and "why", not detailed "how"
- Include production impact metrics when available
- Use absolute URLs (base: https://aitutorial.dev)

### llms-full.txt (Complete Version)

**Purpose**: Comprehensive reference with full content — for deep dives, code generation, and detailed answers.

**Target size**: No limit, but must be well-structured with clear navigation.

**Content structure**:
```markdown
# AI Tutorial.dev - Complete Documentation
> [Site description]
> Last Updated: [git commit hash] [timestamp]
> Site: https://aitutorial.dev

## Table of Contents
[Hierarchical listing with absolute URLs]

---

## Module Name
[Full module overview content]

### Section Name
[Complete section content including all text, code blocks, examples]

---
```

**Guidelines**:
- Include full explanations and context
- Preserve all code examples with language tags (typescript, python, bash)
- Maintain production examples and real-world metrics
- Keep detailed implementation patterns and decision trees

## Content Processing Rules

1. **Follow docs.json order**: The navigation structure in `docs.json` defines the canonical ordering
2. **Clean MDX syntax**: Remove imports, JSX components (Card, CodeGroup, CodeEditor, LLMPlayground, etc.) — convert to plain markdown equivalents
3. **Preserve code blocks**: Keep language tags and full code content
4. **Absolute URLs**: Convert all relative links to absolute (base: `https://aitutorial.dev`)
5. **Normalize headings**: Use standard markdown (#, ##, ###)
6. **Strip frontmatter**: Remove YAML frontmatter but preserve title and description
7. **Section separators**: Use `---` between major sections
8. **No duplication**: Each piece of content appears once

## Interactive Components

The site uses interactive components that need special handling in llms.txt files:

| Component | In llms.txt | In llms-full.txt |
|-----------|-------------|------------------|
| `<CodeEditor>` | Mention the example file path | Include the referenced source code inline |
| `<LLMPlayground>` | Describe what it demonstrates | Include the prompt/system message as a code block |
| `<Card>` | Convert to a bullet point | Convert to a heading + paragraph |
| `<CodeGroup>` | Pick the primary language | Include all language variants |

## Update Process

1. Run `git log -1 --format="%H %ci"` to get latest commit info
2. Read `docs.json` for current navigation structure
3. Process each MDX file in docs.json order
4. Generate `llms.txt` with summaries
5. Generate `llms-full.txt` with complete content
6. Validate both files are well-formed markdown

## Maintenance Checklist

When regenerating files:
- [ ] Git commit timestamp updated
- [ ] `docs.json` navigation structure followed exactly
- [ ] All MDX files processed in order
- [ ] Code examples preserved with correct language tags
- [ ] Tables formatted correctly
- [ ] All URLs are absolute (https://aitutorial.dev/...)
- [ ] No MDX/JSX syntax remaining
- [ ] Section separators (---) between major sections
- [ ] Both files validated as proper markdown
- [ ] Typo in "Leassons" fixed to "Lessons" (legacy issue)
